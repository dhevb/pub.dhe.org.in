# SEO Architecture

Search engine optimization for Viksit Bharat Journal — built for Google Scholar, international indexing, and multilingual discovery.

---

## Canonical URL

**Source:** `src/lib/site-url.ts` → `SITE_URL`  
**Default:** `https://pub.dhe.org.in`  
**Override:** `NEXT_PUBLIC_SITE_URL`

---

## Metadata Generation

### Core helper

`src/lib/seo/metadata.ts`:

| Export | Purpose |
|--------|---------|
| `siteConfig` | Name, description, publisher, keywords, URL |
| `buildMetadata({ title, description, path, noIndex?, ... })` | Next.js `Metadata` object |
| `journalMetadata(journalId, ...)` | Per-journal metadata |
| `journalSectionMetadata(...)` | Journal sub-pages |

Every page exports `metadata` or `generateMetadata` using these helpers.

### Root layout

`src/app/layout.tsx` — `buildMetadata({})` + JSON-LD for Organization and WebSite.

---

## Sitemap

**Route:** `src/app/sitemap.ts`  
**Builder:** `buildSitemapEntries()` in `src/lib/seo/sitemap-urls.ts`

Includes:
- Portal routes (`/`, `/about`, `/search`, `/conferences`, etc.)
- Journal entry routes (`/vbe`, `/vbh`, `/vie`, `/vih`)
- Legacy journal sections (`/vbe.rase/SubmitManuscript`, etc.)
- All 20 paper routes (`/vbe.rase/Paper1` … `/vih.rase/Paper5`)

**URL:** `https://pub.dhe.org.in/sitemap.xml` (~83 URLs)

## Robots

## Structured Data (JSON-LD)

**Component:** `src/components/seo/JsonLd.tsx`  
**Schemas:** `src/lib/seo/schemas.ts`

| Function | Schema.org type | Used on |
|----------|-----------------|---------|
| `organizationSchema()` | Organization | Root layout |
| `websiteSchema()` | WebSite | Root layout |
| `journalSchema(journal)` | Periodical | Journal pages |
| `scholarlyArticleSchema(paper, journal)` | ScholarlyArticle | Paper pages |
| `personSchema(author)` | Person | Author profiles |
| `faqSchema(faqs)` | FAQPage | Homepage |
| `breadcrumbSchema(items)` | BreadcrumbList | Nested pages |
| `bookSchema(...)` | Book | Books section |
| `eventSchema(...)` | Event | Conferences |
| `datasetSchema(...)` | Dataset | Research data |

Injected via `<JsonLd data={[...]} />` in page components.

## Google Scholar

**Module:** `src/lib/seo/google-scholar.ts`

`googleScholarMetadata(data, options)` generates Next.js `Metadata.other` with `citation_*` meta tags:

| Tag | Source |
|-----|--------|
| `citation_title` | Paper title |
| `citation_author` | Authors array |
| `citation_publication_date` | Published date |
| `citation_journal_title` | Journal name |
| `citation_issn` | `2278-1757` from `content/site.json` |
| `citation_pdf_url` | PDF link if available |
| `citation_doi` | Via `buildArticleDoi()` when DOI prefix configured |

**Used in:** `src/components/journal/PaperPage.tsx` → `generatePaperMetadata()`

### QA verification

```bash
node scripts/qa/verify-scholar.mjs
# Or with custom URL:
BASE_URL=https://pub.dhe.org.in node scripts/qa/verify-scholar.mjs
```

Checks 7 required Scholar tags + optional DOI, volume, issue, pages.

## SEO Checklist per New Page

1. Export `metadata` via `buildMetadata({ path: "/your-path" })`
2. Add route to `src/lib/seo/sitemap-urls.ts` if public
3. Add to `scripts/qa/routes.mjs` smoke list if critical
4. For papers: ensure `googleScholarMetadata()` in `generateMetadata`
5. Run `npm run qa:smoke` after deploy
